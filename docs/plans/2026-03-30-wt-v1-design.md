# wt v1 — Design

## Architecture

Subcommand dispatch pattern. `bin/wt` is the entry point, dispatches to scripts in `commands/`. Shared logic lives in `lib/`. Shell wrappers handle `cd`.

```
bin/wt                     # entry point: arg parsing, dispatch
commands/cd.sh             # create-or-switch to worktree
lib/config.sh              # config loading (file + env overrides)
lib/hooks.sh               # hook discovery and execution
lib/git.sh                 # git helpers (find main branch, repo root, etc.)
shell/wt.bash              # bash shell function
shell/wt.zsh               # zsh shell function
```

## Core command: `wt <name> [--from <branch>]`

Default command. Dispatches to `commands/cd.sh`.

1. Load config
2. Resolve base path (default: `<repo-root>/.worktrees`)
3. Check if `.worktrees/<name>` exists
   - **Exists:** run switch hooks, print path to stdout
   - **Doesn't exist:**
     a. Determine base branch (`--from` flag > `WT_BASE_BRANCH` config > auto-detect main/master)
     b. `git worktree add .worktrees/<name> -b <name>` (or without `-b` if branch already exists)
     c. Run create hooks
     d. Print path to stdout

Explicit form: `wt cd <name>` — identical behavior.

All messages go to **stderr**. Only the worktree path goes to **stdout**.

## Shell integration

Thin function sourced in user's rc file:

```bash
wt() {
    local result
    result="$(command wt "$@")"
    local rc=$?
    if [ $rc -eq 0 ] && [ -d "$result" ]; then
        cd "$result"
    elif [ -n "$result" ]; then
        echo "$result"
    fi
    return $rc
}
```

**Install:** Add `bin/` to PATH + `source shell/wt.bash` (or `.zsh`) in rc file. Symlink or PATH modification for development.

## Hooks

Two hook directories under `.worktrees/hooks/`:

- `.worktrees/hooks/create/` — runs after worktree creation
- `.worktrees/hooks/switch/` — runs when switching to an existing worktree

Rules:
- Scripts run in lexicographic order
- Must be executable
- Receive worktree path as `$1`
- Non-zero exit prints warning to stderr, continues
- Missing hook dirs are silently skipped

## Config

File: `.worktrees/config` — shell-sourceable key=value.

```
WT_BASE_BRANCH=main
WT_BASE_PATH=.worktrees
```

Both optional. Env vars override file values. Defaults: auto-detect main/master, `.worktrees`.

## Error handling

- Not in a git repo → error, exit 1
- No name provided → usage message, exit 1
- Branch exists but no worktree → `git worktree add` using existing branch (no `-b`)
- Stale worktree path → error suggesting cleanup
- Hook failures → warning, continue

## Edge cases

- `.worktrees/` created on first use
- Name conflicts with subcommands → subcommands win; use `wt cd <name>` to disambiguate
- `.worktrees/config` missing → defaults
- Hook dirs missing → skip silently
